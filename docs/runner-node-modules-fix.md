# Runner node_modules Corruption Fix

## Problem

The sync process was treating `node_modules/` as source-of-truth files and removing them as "stale" during sync operations. This caused:

- **"removed 13552 stale" log messages** that deleted critical dependency files
- **Broken binary dependencies** like `esbuild` and `vite`
- **Failed npm install attempts** that tried to repair corrupted `node_modules` instead of doing clean installs

## Solution

Three changes were implemented:

### 1. Exclude node_modules from sync entirely

**File:** `preview-runner/index.js`

- Added `node_modules/` to the sync exclusion list at ALL levels (not just root)
- These are ephemeral build artifacts generated by the runner, NOT source files
- They should NEVER be written to the database or removed as "stale"

**Verification:** Runner logs now show:
```
[sync] node_modules excluded from sync (ephemeral build artifacts, not source files)
```

### 2. Add retry logic to npm install

**File:** `preview-runner/index.js` → `runInstallIfNeeded()`

When `npm install` exits with code 1:
1. Log the failure clearly
2. Delete the entire `node_modules/` directory
3. Retry the install command once from scratch
4. If the retry fails, surface the error to the user with the full npm log

**Verification:** Runner logs show:
```
[runner] npm install failed (attempt 1/2): npm install exited 1
[runner] deleting corrupted node_modules and retrying from scratch…
[runner] deleted /project/frontend/node_modules
[runner] running npm install in /project/frontend (this may take 1-2 min on cold start)…
```

### 3. Database cleanup endpoint

**Endpoint:** `POST /api/previews/:projectId/cleanup-node-modules`

Removes all `node_modules` rows from the `project_files` table for a given project.

**Usage:**

```bash
# Via API (requires auth)
curl -X POST https://auroraly.co/api/previews/e5e4f1f4-3b5e-4c55-b2dc-655f483ef3ed/cleanup-node-modules \
  -H "Authorization: Bearer YOUR_TOKEN"

# Via script (requires env vars)
NEXT_PUBLIC_SUPABASE_URL=... \
SUPABASE_SERVICE_ROLE_KEY=... \
node scripts/cleanup-node-modules-from-db.js e5e4f1f4-3b5e-4c55-b2dc-655f483ef3ed
```

**Response:**
```json
{
  "ok": true,
  "deleted": 13552,
  "message": "Deleted 13552 node_modules rows. Preview machine stopped. Next start will do a fresh npm install."
}
```

## Prevention

Going forward:

- **Never sync node_modules** — the runner's sync exclusion list now catches them at all levels
- **Retry on install failure** — the runner automatically recovers from corrupted `node_modules`
- **Clean database** — the cleanup endpoint is available for emergency recovery
